`Q: What is NUMEN ?`



NUMEN is an Application as Service Cloud Platform. The application which enables the users to experience the Amazon Web Services along the Scientific Applications in the most efficient way,

additionally it provides the users with an interactive UI.



`Q: Why to choose NUMEN ?`



NUMEN enables users to use Prebuilt, auto-deployed, and Optimised Scientific Applications which reduces the efforts and time spent to Install, Integrate, Configure and Maintain the Infrastucture as well as Applications like, CryoSPARC, Relion, Gromacs and Alphafold2.



`Q: Which all applications Does NUMEN supports ?`



As of Now, NUMEN version 1.0 Contains Scientific Applications integrated with it.

Current Version of NUMEN version 1.0 has Gromacs, CryoSPARC, Relion3, Relion4 and Alphafold2, which are Optimised and pre-built with the AWS Infrastructure platform.

If you need any scientific application to be integrated and avilable please reach out to our support team on - `numen-support-sun@clovertex.com`.



`Q: What are the benefits of using NUMEN in research ?`



NUMEN provides prebuilt applications for Multiple Scientific tools which are provided using AWS cloud platform (Infrastructure), which willl be available on few clicks, reduces the Efforts and saves Scientist's/Researcher's Or user's time to build the Infrastructure and hectic deployments, so that they can directly test their simulations and research activities over it.



`Q: How Does NUMEN works ?`



As the NUMEN web ui, provides an automated, click based Infrastructure deployment feature, all we need to do is, to select the infrastructure components as per the requirement.

The web Ui contains the icons with respect to the infrastructure type such as Applications, Visualization tools, Packages, OS, Data Sources, and Infrastructure.

This ‘NUMEN’ console has a feature called as ‘Express launch’ which provisions applications, as soon as the user selects an application by clicking on it, the console will also select other components as per the ‘preselect’ which is already built in by default.

eg : If user clicks on ‘CryoSPARC’ application, other infra components will get auto select by default. If you want the custom changes, NUMEN provides that option too in 'Custom Launch'.



`Q: Can we customise the Application Infrastructure as per our need ?`



Yes, NUMEN has the feature called 'Custom Launch', which will provide multiple options to customise the application infrastructure using number of infrastructure relevant options.

If the default selection is not as per the requirements, users are having privilege to change the selections, and can opt for the other options. Once the selection is done with respect to the requirements of the users, user need to click n the ‘launch’ button.



`Q: Does NUMEN supports clustering on Applications ?`



Yes, NUMEN supports the stand alone as well as Clustering on Applications, as of now applications can be deployed using AWS Paralell cluster 3.2.1 by default.



`Q: Does NUMEN contains any sample data ?`



Yes, NUMEN contains pre integrated datasourcess which do contain the different datsets from different CROs.



You can find these datsets under `/data` as well as `/projects`.



`/data` is the read only datasets which is the dump of the data from different CROs.



`/projects` is the place holder where users can create their own projects and can store the processed data as well.



`Q: How can one get access to the NUMEN ?`



NUMEN is deployed on AWS using ALB services, you have been provided with a script which is to be used to update or whitelist your IPs, once its updated you can access the NUMEN console with the link which is shared with you via mail.

`Q: How to access the instances using SSH ?` 



As of now NUMEN does not provide any privilege to access any instance using SSH, for Single Nodes or Head Nodes.
But for Parallel Cluster `Compute Nodes` YES, if you copy your SSH public key from your laptop to .ssh/authorized_keys if we keep the environment in public network, you will be able to SSH to that perticular instance.
